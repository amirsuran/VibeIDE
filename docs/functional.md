# Возможности VibeIDE

Полный каталог функциональности VibeIDE — что умеет редактор и его AI-ядро. Документ
организован по темам, без привязки к номерам версий: здесь — «что есть», а история
изменений по релизам живёт в [GitHub Releases](https://github.com/VibeBrains/VibeIDE/releases).

> Новое в каждом релизе попадает сюда автоматически — при сборке ченджлога секция «✨ Новое»
> сливается в соответствующую тему этого файла (см. `CLAUDE.md` → «Версионирование»).

## 🧠 AI-ядро

- **Роутер LLM-провайдеров** — OpenAI, Anthropic, Gemini, Mistral, Ollama и другие. Разные модели для разных задач (чат, автокомплит, inline-правки).
- **Свои LLM-провайдеры через `.vibe/providers.json`** — добавить, переопределить или выключить провайдера и модели **без пересборки IDE**. Работают как встроенные: карточка в «Облачных провайдерах», проверка валидности ключа (✓/✗), источник ключа (`.vibe/.env` или введён в IDE), живой каталог моделей из `<baseURL>/v1/models`, вкладка «Модели» с тумблерами. Возможности модели (vision, reasoning, формат инструментов, размер контекста) берутся из базы знаний так же, как у встроенных. Патч встроенных по `id`, клон через `extends`, порядок через `order` + IntelliSense и диагностика в редакторе.
- **Локальные модели (Ollama, LM Studio)** — автодетект и онбординг встроены.
- **Автокомплит FIM (Fill-in-the-Middle)** — встроенный, без дополнительных расширений.
- **Инлайн-редактирование кода** (Ctrl+K), детекция ошибок и код-ревью силами выбранной модели. Ctrl+K понимает слэш-команды `/doc`, `/refactor`, `/tests`, `/explain`, `/fix`.
- **Индексация кодовой базы и семантический поиск (RAG)** по всему репозиторию.
- **Image QA** — анализ скриншотов и диаграмм как часть контекста (с privacy-гейтом и лимитом размера).

## 🤖 Агенты и автопилот

- **Мультиагентная оркестрация** с территориальными блокировками — агенты не мешают друг другу.
- **Персистентные планы** — план агента переживает перезапуск IDE и продолжается с места остановки; финал плана записывается на диск, без ложного «Continue Plan» на каждом старте. **План не паузится на синонимах инструментов:** `edit_file` ↔ `rewrite_file` и `run_terminal_command` ↔ `run_command` считаются одним классом («правки» / «терминал»), а при включённом автопилоте план продолжает выполнение даже при инструменте вне списка шага (с инфо-уведомлением; строгость настраивается `vibeide.plans.toolDriftPause`: always / manual-only / never). Если шаг всё же на паузе — у поля ввода появляется чип «⏸ План на паузе — возобновить» (плюс кнопка-стрелка к карточке плана) — не нужно листать чат вверх.
- **Skills system** — переиспользуемые AI-поведения на уровне проекта; `/skill:NAME` выгружается в контекст целиком.
- **Автопилот** — агент доводит шаг до конца сам, без выпрашивания «продолжить»; на длинной серии шагов делает чекпоинт-вопрос «продолжать?». Завершает ход через `vibe_complete` вместо текстового «готово» в цикле.
- **Детектор петель** — автоматически останавливает зациклившегося агента.
- **Dead man's switch** — защитный механизм для долгих автономных задач.
- **Самовосстановление** — IDE пережидает лимиты провайдера, перезапускает зависшие ходы и переключает формат инструментов (native ↔ XML) на лету.
- **Роли-субагенты (Vibe Agents)** — набор ролей (планировщик, дизайнер, фронтенд/бэкенд, ревьюер, QA, security) с правами на роль: ревьюер и security работают только на чтение, без записи. Каждая роль исполняется настоящим изолированным агентом: свой транскрипт, **независимый токен-бюджет и своё контекстное окно** (настройка `vibeide.subagent.maxTokens`, не зависит от расхода основной сессии; учёт — по реальным данным провайдера, кэшированная история повторно не тарифицируется; токены и контекст роли считаются отдельно и НЕ приписываются к индикаторам «Сессия»/«Контекст» основного треда), инструменты строго по списку роли, а перед записью файлов или запуском команд роль спрашивает разрешение — молча ничего не меняет (гейты одобрения **наследуются от основного агента, ровно те же**: при включённом автопилоте роль выполняет инструменты без запроса, как и основной агент; иначе — по per-type авто-одобрению или с диалогом); родитель получает только компактный итог; отменённая роль немедленно прекращает работу, включая уже отправленный запрос к модели. **Модель на роль:** в настройках (секция «Роли агентов») каждой роли можно назначить свою модель — read-only роли выгодно сажать на лёгкую (дешевле и быстрее, а писать код им всё равно запрещено); по умолчанию роли работают на модели чата. Команда «Показать маршрут ролей» подсказывает, кто и в каком порядке возьмётся за задачу, а «Выполнить маршрут ролей» — запускает команду ролей по задаче: этапы идут по порядку (роли внутри этапа — параллельно), прогресс виден в статус-баре субагентов (клик — отменить роль) **и прямо в чате**: пока роль работает, в треде крутится живой индикатор «🧩 Роль «…» работает… (шаг 12/40 · ~24k)» — показывает связывающий лимит (шаги — обычно роль упирается в них раньше токенов) и расход токенов роли, а по завершении остаётся компактный итог роли (успех/остановка/пропуск с токенами и ≈$); всё это включается настройкой `vibeide.subagent.chatNotices`; по завершении **сводный отчёт приходит сообщением в чат запуска** — с итогами каждой роли, токенами, моделью и примерной стоимостью в $; security подключается автоматически на чувствительных темах (OAuth, платежи, секреты). **Лимит шагов на прогон** роли задаётся настройкой `vibeide.subagent.maxSteps` (дефолт 60) — для слабых моделей, жгущих шаги на чтении, поднимайте. **Под автопилотом ресурсные лимиты роли (шаги/время/токены) авто-продлеваются**, и роль НЕ встаёт на паузу — идёт до готовности или отмены, как основной агент под автопилотом (никаких новых стопов от субагентов); лимиты снова вступают в силу, если выключить автопилот. **Сохранение прогресса (durable handoff)** действует в ручном режиме: роль, упёршаяся в лимит, не проваливается — частичный результат сохраняется (переживает даже перезапуск IDE), и роль автоматически продолжается с места остановки с увеличенным бюджетом; число авто-продолжений задаётся пилюлей **«субпин.»** прямо в тулбаре чата (рядом с «подпин.», ключ `vibeide.subagent.maxResumes`; субагентный аналог авто-нуджа основного агента), плюс `vibeide.subagent.resumeBudgetFactor` для роста бюджета; когда авто-продолжения исчерпаны, **в самом чате** появляется кнопка **«Продолжить роль»** (там же, где чатовская «Продолжить») со счётчиком ожидающих ролей — клик открывает выбор роли и продолжает её вручную с сохранённого места; отказы пользователя уважаются: отменённая роль и роль с многократно отклонёнными инструментами автоматически не перезапускаются.
- **Дисциплина минимализма кода** — режимы `off`/`лайт`/`фулл`/`ультра` (настройка «Минимализм агента» в разделе «Инструкции для ИИ» или ключ `vibeide.global.minimalismMode`). Лайт — «сначала переиспользуй»: существующий код → stdlib → платформа → установленные зависимости, без спекулятивных абстракций (YAGNI). Фулл — полная лестница из 7 ступеней перед написанием любого нового кода. Ультра — плюс требование минимального ревьюабельного диффа: агент оспаривает избыточные требования и явно перечисляет, что сознательно не реализовал. Валидация, обработка ошибок, безопасность и доступность не урезаются ни в одном режиме; правила проекта (`.vibe/rules`, `AGENTS.md`) приоритетнее дисциплины. Меньше кода — меньше токенов и быстрее ход.
- **Делит-лист и леджер упрощений** — слэш-команда `/simplify` в чате прогоняет текущий git diff по лестнице минимализма и возвращает делит-лист (файл:строка, что удалить/упростить, почему, экономия строк), ничего не меняя в файлах. Сознательно отложенные упрощения помечаются в коде комментарием `vibe-later: <что и почему>` (агент делает это сам в режимах фулл/ультра); команда палитры «Леджер отложенных упрощений (vibe-later)» собирает все маркеры воркспейса в markdown-отчёт с кликабельными `путь:строка`.

## 🌐 Vibe Server — локальный предпросмотр

- **Панель «Vibe Server»** поднимает локальный сервер и показывает результат прямо в IDE (встроенный браузер) или во внешнем браузере — на выбор.
- **Три режима под проект:** статический сайт (HTML/CSS/JS), **dev-сервер фреймворка** (Vite/Next/React/Vue/Angular — запускается ваш `npm run dev` и берётся его URL) и **Docker-окружение** (`docker compose` / Dockerfile — кнопкой «Поднять окружение»).
- **Живая перезагрузка:** правка файла сразу обновляет превью; CSS применяется без перезагрузки страницы (состояние и прокрутка сохраняются).
- **Встроенный браузер** с адресной строкой, навигацией, размерами под телефон/планшет/десктоп и поворотом; несколько вкладок и синхронизация прокрутки. **Cookie-логин на превью работает:** пока вкладка показывает localhost-сайт, его куки автоматически дополняются `SameSite=None; Secure` — иначе браузер отбрасывал бы их в cross-site iframe и авторизация молча ломалась; действует только на loopback и только при открытом превью, отключается `vibeide.vibeServer.cookieCompat`.
- **Превью на телефоне** — адрес в локальной сети (LAN) и **QR-код** для быстрого открытия.
- **Ошибки превью в чат** — ошибки консоли и проваленные сетевые запросы одной командой уходят агенту; счётчик ошибок виден в строке состояния.
- **Без зомби-процессов** — dev-сервер корректно гасится при закрытии IDE (kill по владельцу порта), а осиротевший от прошлого запуска выметается на старте; новый сервер не уезжает на чужой порт.
- **Занятый порт проекта — под контролем.** Если порт из конфигурации проекта занят (dev-сервер ушёл на соседний порт или упал с EADDRINUSE), IDE показывает модальное окно: **освободить порт** (завершить занявший его процесс и перезапустить dev-сервер), **работать на подменённом порту** (только на эту сессию — порт в конфигурации проекта не меняется никогда, деплой ждёт именно его) или **отмена** (разрулить самому). Если порт держит dev-сервер другого проекта, запущенного в этой же VibeIDE, окно называет проект и по умолчанию предлагает сосуществование, а не убийство соседа. Работа на подменённом порту помечается в статус-баре: `3001 (вместо 3000)`. Отключается настройкой `vibeide.vibeServer.portConflictPrompt`.

## 💬 Чат и контекст

- **Подмешивание контекста на ходу** — пока агент работает, набранный текст по Enter встаёт в очередь и учитывается на следующем шаге, без остановки хода. Очередь видна **закреплённой полоской над полем ввода**, любую заметку можно убрать крестиком.
- **Закрепление сообщений булавкой** — важное в чате не вырезается при обрезке истории.
- **Динамическая фильтрация контекста** — промежуточные результаты инструментов сжимаются до попадания в основной контекст (режимы `auto`/`raw`/`aggregate`/`off`), с явным маркером усечения — без тихого удаления.
- **Кеширование промпта** — повторные большие префиксы контекста стоят кратно дешевле.
- **Таймлайн чата** — видно, где ход «задумался», без копания в консоли; у инструментов виден `start`/`done` и длительность.
- **Правила проекта** — дерево `.cursor/rules/` и legacy `.cursorrules` подхватываются и ходят по ссылкам (`@rule`), как в Cursor.
- **Экспорт чата в Markdown** — скопировать переписку в буфер обмена (tool-результаты усечены для компактности) или сохранить **полный** лог в `.md`-файл через диалог сохранения; свёрнутые блоки рассуждений сериализуются независимо от состояния UI.
- **Автоскролл и кнопка «вниз»** — пока агент стримит, чат сам держится у низа (с запасом, без отрыва на мелком отставании); если отлистал вверх читать историю — внизу появляется кнопка прокрутки к последнему сообщению.

### Лимиты сессии и контекстного окна

- **Session token limit** — потолок input+output токенов на чат-сессию (по умолчанию 2 000 000). Warning на 80%, блокировка на 100% до reset; при включённом автопилоте — авто-сброс с записью в лог (rate-limited).
- **Context window guard** — следит за заполнением окна модели в реальном времени: non-blocking warning на 75%, blocking-диалог *compact / continue / cancel* на 90%.
- **Индикатор в статус-баре** — текущее заполнение контекста, цвет зелёный → жёлтый → красный; по клику открывается status-pane. Показывает реальные токены провайдера.
- **Per-task split (опционально)** — делит session-budget между задачами очереди агентов пропорционально.

## ✏️ Надёжные правки файлов

- **Простой формат `edit_file`** — «что заменить» (`old_string`) и «на что» (`new_string`), два обычных поля. Слабые модели (MiniMax и др. через агрегаторы) больше не сваливаются в десятки одноразовых скриптов.
- **Терпимый поиск** заменяемого текста — правка применяется, даже если модель чуть ошиблась в отступах/пробелах или слегка перефразировала середину (line-trimmed + block-anchor).
- **Выравнивание отступа** блока по файлу, если модель скопировала якорь без отступа.
- **Без потери кода** — незавершённый блок правки отклоняется с подсказкой, а не применяется как удаление.
- **Режим «Обзор»** (read-only) не зацикливается — агент прямо предлагает переключиться на «Агента» для применения правок.
- **Рефактор-инструменты** (rename/extract/generate_tests) — под тем же подтверждением, что и обычные правки.

## 🔔 Уведомления о фоновой активности агента

Когда агент завершил работу, остановился или ждёт вашего ответа, а IDE свёрнута или в фоне:

- **Звук** — короткий сигнал на завершение / остановку / ожидание ответа. Раздел «Уведомления» в настройках: выбор звука с прослушиванием, свой файл через «Обзор…» (mp3/ogg/wav, до 1 МБ и 5 секунд — проверяется при выборе), громкость, «молчать, когда окно в фокусе», тумблеры событий.
- **Редактор «VibeIDE Звуки»** — загрузить свой трек, обрезать окно до 5 секунд по звуковой волне, прослушать и сохранить (в папку `sounds` рядом с данными IDE).
- **Мигание иконки в панели задач** (на macOS — прыжок в Dock) — не отвлекает, но забыть не даёт; клик возвращает к IDE и переключает на её рабочий стол.
- **Системный тоаст** с тем же сообщением; клик по нему тоже возвращает к VibeIDE.

Всё срабатывает только когда окно не в фокусе. Отключается тумблером.

## 🔌 Провайдеры: диагностика и надёжность

- **Модалка «Проверка провайдеров»** — показывает активные провайдеры (у кого прописан ключ) и послойно проверяет связь: конфиг → сеть → авторизация → каталог моделей, с задержкой ответа. Пятый слой — **сквозной тест запроса (L5)**: по кнопке (никогда автоматически — тратит токены) уходит настоящий мини-запрос к модели через тот же путь, что и чат; меряется время первого токена и полного ответа — ловит «залипший транспорт», который послойная проверка не видит. Если слои зелёные, а сквозной тест падает, модалка сама предлагает «Сбросить клиентов». Тумблер «Авто» перепроверяет слои каждые 30 секунд, пока окно открыто (L5 не трогает); кнопка «⧉» на карточке копирует отчёт по одному провайдеру. Отчёт выгружается в Markdown вместе с журналом событий отправки (синк провайдеров, кэш клиентов, пересоздание соединения, первый токен, ошибки; секреты вычищаются). Индикатор здоровья провайдера в статус-баре при деградации открывает эту модалку по клику.
- **«Починить связь»** — иконка-ключ в статус-баре лечит «токены вдруг перестали приходить у всех провайдеров» **без перезапуска IDE**: пересоздаёт сетевое соединение и сбрасывает кэши клиентов. Плюс **автолечение**: если сетевые сбои идут сразу у двух и более провайдеров в одном окне (сигнатура залипшего общего пула соединений), IDE пересоздаёт транспорт сама — с уведомлением и кнопкой «Проверка провайдеров»; не чаще раза в 10 минут, отключается `vibeide.llm.autoResetTransport`. Смена настроек провайдера (включая заголовки в `.vibe/providers.json`) теперь подхватывается без перезапуска — клиент пересоздаётся автоматически.
- **Failover провайдеров** — переключение при недоступности.
- **Внешний каталог квирков моделей** — таблица особенностей моделей (пустой ответ при стандартной `temperature`, требование `reasoning_content`, XML-режим вместо native function-calling) берётся из самого свежего источника (рядом с exe / из сборки / с CDN, по дате). Новая «странная» модель — это PR на один файл, а не релиз IDE. **Каталог сам просится пополниться:** если модель из сессии в сессию уезжает в авто-даунгрейд из-за сломанных native tool-call, IDE один раз предложит закрепить XML-режим насовсем (кнопка в уведомлении) и скопировать готовое правило для PR в каталог — квирк станет дефолтом для всех. Порог и выключатель — `vibeide.modelQuirks.autoSuggest*`.
- **Нормализация вызовов инструментов** — имена `file_read`/`FileRead`/`ReadFile` сводятся к одному концепту; аргументы от чужого тула чинятся на лету; self-closing-теги (`<read_file path="..." />`) распознаются. В настройках (Безопасность → «Нормализация tool-call») — **живые счётчики слоёв защиты**: видно, какая форма «нестандартных» вызовов у текущей модели и какой слой её тащит; кнопки «Обновить» и «Сбросить счётчики» для А/Б-проверки конкретной модели.
- **Нативные режимы рассуждений моделей** — слайдер «мышления» в чате говорит с моделью на её родном языке: для MiniMax-M3 это `adaptive` (модель сама решает, думать ли — дефолт, экономит токены на тривиальных шагах) / `enabled` (думать всегда) / выключено; для Anthropic — бюджет токенов; для OpenAI-совместимых — уровни усилия.

## 🔗 MCP (Model Context Protocol)

## ⌨️ Команды проекта

- Описываются один раз в `.vibe/commands.json` — едут в репозитории вместе с проектом, у коллег появляются автоматически.
- Закреплённые (`pinned: true`) — отдельное меню **Команды** в шапке IDE: inline pin / edit / delete без перехода в настройки.
- Топ-9 закреплённых — горячие клавиши `Ctrl+Shift+Alt+1..9` автоматически (пере-биндятся при изменении набора).
- Импорт из `.vscode/tasks.json` через палитру одним кликом.
- Sanitizer режет shell-метасимволы и zero-width injection; первый запуск и любая правка команды требуют явного approve (hash в `.vibe/commands.trust.json`).
- Секреты через `${secret:KEY}` — реальные значения не попадают в audit-лог и не уходят в облачные индексаторы.
- Редактор: форма с валидацией или прямой JSON-режим.

## 🔒 Безопасность и приватность

- **Нет телеметрии Microsoft** — поле `enableTelemetry` убрано из `product.json`. Локальное хранение телеметрии с явным opt-in для облака.

- **Аудит-лог** — `.vibe/audit.jsonl`, одна служебная папка в проекте.
- **First-run security wizard** и анализ разрешений расширений.

## 🩺 Диагностика и стабильность

- **Watchdog памяти** — следит за ростом каждые несколько минут, предупреждает до OOM (pre-OOM detector), умеет собрать crash-report и снять heap snapshot.
- **Восстановление после краша** ext-host и stale execution lease — баннер «взять управление / отменить» вместо немого зависания.
- **OTLP-трейсы агентного цикла** — spans на tool-calls, latency провайдера, размер контекста (локальный экспорт, не облако по умолчанию).
- **Сырые XML-теги в чате** прячутся в лаконичный плейсхолдер вместо протечки в UI.
- **Копирование логов одной кнопкой** — диагностические/агентские логи берутся в буфер без выковыривания из файлов.
- **Единое управление логированием** — настройки `vibeide.logging.*` (уровень, категории с wildcard, per-category уровни, схлопывание повторов) действуют живьём на **все процессы**, включая main; редакция секретов работает во всех приёмниках лога каждого процесса (консоль, ring-buffer → буфер обмена, Output-канал, файл).

## 📦 Установка и сборка

- **Установщик `VibeIDESetup.exe`** (издатель VibeBrains) + portable `.zip`.
- **macOS (Apple Silicon, arm64)** — `.dmg` (перетаскиванием в Applications) + портативный `.zip`. Сборка пока не нотаризована Apple: при первом запуске Gatekeeper требует «Open Anyway» (System Settings → Privacy & Security) либо `xattr -cr /Applications/VibeIDE.app`; SHA-256 артефактов публикуются в release notes. Intel-маки не поддерживаются.
- **Доступен в winget** — `winget install VibeBrains.VibeIDE` (Windows).
- **Портативный режим** — папка `data/` рядом с исполняемым файлом.
- **Двухфазный релиз-поток** (сборка → ручная проверка → публикация) — для самого проекта, на обеих платформах (`release-windows.ps1` / `release-macos.sh`).
