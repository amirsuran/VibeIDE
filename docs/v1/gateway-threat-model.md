# Gateway Threat Model

> Обязателен до М-Фазы 0 (запуска Gateway). Gateway не запускается без этого документа.

---

## Что такое vibe-gateway

Облачный прокси к LLM-провайдерам (OpenAI, Anthropic, etc.).  
Пользователь платит через gateway вместо ротации собственных ключей.

**Важно:** Gateway — только прокси к LLM. Синхронизация `.vibe/` файлов идёт через VSCodeSyncFiles в облако **пользователя**, не через gateway.

---

## Что хранится на сервере gateway

| Данные | Хранится? | Срок |
|---|---|---|
| Промпты пользователей | ❌ Нет | — |
| Код из контекста | ❌ Нет | — |
| Метаданные запроса (модель, токены, timestamp) | ✅ Да | 30 дней для billing |
| Идентификатор пользователя | ✅ Да (hashed) | Пока активна подписка |
| IP-адрес | ✅ Да | 7 дней (rate limiting) |

**Принцип:** gateway видит только routing metadata, не содержимое промптов.

## Attack Surface

### 1. MITM между IDE и gateway
**Угроза:** перехват промптов пользователя.  
**Защита:** TLS 1.3 обязателен. Certificate pinning в Electron — рассмотреть при М-Фазе 1.

### 2. MITM между gateway и LLM-провайдером
**Угроза:** подмена ответов модели.  
**Защита:** TLS 1.3. Провайдеры используют HTTPS. Fingerprinting ответов — backlog.

### 3. Компрометация gateway (single point of failure)
**Угроза:** атакующий получает доступ к routing metadata всех пользователей.  
**Защита:** 
- Минимальный набор хранимых данных
- Rotation API keys не хранятся на gateway-стороне
- Incident response: немедленный сброс всех сессий, уведомление пользователей

### 4. Abuse / Rate limiting
**Угроза:** зацикленный агент = неограниченные расходы через gateway.  
**Защита:** hard limits на уровне gateway (отдельные от IDE-лимитов), configurable per-user budget.

### 5. API Key exposure на стороне gateway
**Угроза:** gateway использует мастер-ключ для роутинга.  
**Защита:** ключи провайдеров хранятся в secrets manager (не в env vars). Rotation каждые 90 дней.

## Политика логирования

| Что | Логируется? |
|---|---|
| Содержимое промпта | ❌ Никогда |
| Содержимое ответа | ❌ Никогда |
| Модель, токены, latency | ✅ Aggregate only |
| User ID (hashed), timestamp | ✅ Для billing |
| IP-адрес | ✅ 7 дней, rate limiting only |

---

## EU Data Residency (GDPR)

**Решение:** EU-регион gateway — **обязателен** до запуска EU-маркетинга.

- Физическое размещение в EU (Frankfurt / Amsterdam)
- Данные EU-пользователей не покидают EU (даже метаданные)
- Privacy Policy явно указывает регион хранения
- Gateway не запускается без этого для EU-аудитории

---

## Incident Response

1. **Обнаружение компрометации:**
   - Аномалия в billing (резкий рост) → автоматический alert
   - Security scan → manual review
2. **Первые 30 минут:**
   - Сброс всех active sessions
   - Ротация мастер-ключей провайдеров
   - Публичное уведомление на status.vibeide.io
3. **48 часов:**
   - Post-mortem публичный отчёт
   - Уведомление пользователей по email

## Архитектурная сегрегация

```
Privacy-first пользователь:     IDE → собственный API ключ → провайдер
Convenience пользователь:       IDE → vibe-gateway → провайдер
```

Gateway-трек архитектурно изолирован от privacy-трека.  
Пользователь видит в IDE **что именно** уходит через gateway.  
В маркетинге не смешивать.
