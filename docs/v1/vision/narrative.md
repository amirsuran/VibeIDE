# Нарратив и позиционирование

## Нарратив

> **«Ты видишь всё — и управляешь всем»**

Двухсоставной нарратив:
- **«Ты видишь всё»** — точный промпт, контекст, стоимость, reasoning агента
- **«Ты управляешь всем»** — права агента, инструменты, откат любого шага

## Конкурентная позиция

Cursor, Windsurf и аналоги добавляют фичи быстрее, чем любой форк успеет их портировать.  
**Конкурентное преимущество VibeIDE — не паритет фич**, а то, что пользователь *понимает что происходит и может остановить в любой момент*.

### Измеримые дифференциаторы vs Cursor

| Метрика | Cursor | VibeIDE |
|---|---|---|
| Задокументированных параметров fingerprint | 0 | N (публично) |
| Кликов до полного audit log | недоступно | 1 |
| Публичная документация сетевых запросов | нет | Security FAQ |
| Явная атрибуция AI-коммитов в git | нет | Co-authored-by |
| Экспорт и удаление данных (GDPR) | нет | да |
| Публичный incident response guide | нет | да |
| Explicit tool approval per action | нет | да |
| Работает без cloud подписки | нет | да |
| Open-source (аудируемый код) | нет | да |

> Эта таблица **автогенерируется** из `vibe doctor` output и обновляется при каждом релизе.

## Целевая аудитория

1. **Privacy-oriented разработчики** — не устраивает «чёрный ящик» в Cursor
2. **Команды с compliance-требованиями** — fintech, legal, NDA-проекты
3. **Разработчики с требованиями безопасности** — enterprise с корпоративными политиками
4. **Open-source сообщество** — BYOK, локальные модели, нет подписки

## Позиционирование vs Continue.dev

Continue.dev — прямой конкурент с идентичным позиционированием (open-source, BYOK, privacy-first, 25k+ stars).

**Дифференциаторы VibeIDE:**
- **Standalone IDE** (не расширение) — полный контроль над UX
- **Transparency & Control Suite** как первоклассный UX (не просто настройки)
- **Аудит-лог** + **checkpoint infrastructure** — откат любого шага
- **`.vibe/` как открытый стандарт** — `.editorconfig` для AI-агентов

> ⚠️ `vibe init --from continue` обязателен для снижения барьера миграции.

## Trust Score — ключевой UX-артефакт

Постоянный виджет в статус-баре. Не настройка в меню.

| Уровень | Цвет | Поведение |
|---|---|---|
| Manual | 🟢 | Каждое действие требует подтверждения |
| Supervised | 🟡 | Уведомления, автоприменение после таймаута |
| Auto | 🔴 | Агент работает автономно с budget-лимитами |

**UX-требование:** полностью управляется с клавиатуры. Keyboard-first — не опция, а стандарт.
