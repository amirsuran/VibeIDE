# North Star MVP — 10 фич без которых не запускаться

> Минимум для осмысленного первого релиза. Всё остальное — backlog.

| # | Фича | Почему обязательна | Фаза |
|---|---|---|---|
| 1 | **Trust Score виджет** | Центральное выражение нарратива; без него нет идентичности | 1 |
| 2 | **Workspace isolation** | Базовая безопасность; без неё агент опасен | 1 |
| 3 | **Жёсткий дефолтный лимит токенов** | Защита от неожиданных счётов; $20/500k, включён по умолчанию | 1 |
| 4 | **Diff preview + Diff annotations** | Пользователь видит что изменится ДО изменения; core нарратив | 2 |
| 5 | **Dead man's switch + Loop detector** | Агент без контроля — не продукт | 1 |
| 6 | **Debug my prompt** | «Ты видишь всё» начинается отсюда; нет ни у кого из конкурентов | 2 |
| 7 | **Context window visualizer** | Делает невидимое видимым; моментально отличает от Cursor | 2 |
| 8 | **Ollama из коробки** | Privacy-аудитория без этого не придёт | 1 |
| 9 | **Agent git identity** | Compliance-аудитория требует; реализация — один день | 1 |
| 10 | **`vibe doctor`** | Без диагностики невозможна поддержка и доверие | 1 |

> **First-run security wizard** — перенесён в Фазу 1. Без него first-run UX требует ручной настройки.

## Что НЕ входит в MVP

Следующие фичи важны, но **не блокируют первый релиз**:

- Inline diff review → Фаза 2
- Transparency Dashboard → Фаза 2
- Agent action history sidebar → Фаза 2
- CLI (`vibe run --auto`) → Фаза 3a
- Multi-agent режим → Фаза 3b
