# `.vibe/rules.md` — AI-инструкции

## Концепция

Инструкции для агента в Markdown-формате. Агент читает перед каждой задачей.  
Отличие от `constraints.json`: там *что нельзя*, здесь *как работать*.

## Наследование в монорепо

```
project/
├── .vibe/rules.md          ← корневые правила
└── packages/
    └── api/
        └── .vibe/rules.md  ← правила только для api/
```

**Правило приоритета:** ближайший `rules.md` к файлу побеждает.

Варианты (зафиксировать в Фазе 0):
- **Ближайший побеждает** — рекомендуется
- **Merge** — объединяются (опасно при противоречиях)
- **Explicit override** — нижний явно наследует или отменяет верхний

> Риск: #14

## Конфликт с `goals.md`

`rules.md` — *как* агент работает.  
`goals.md` — *что* агент должно достичь.

При обновлении `context.md` агент **автоматически валидирует** изменения против `goals.md`.  
`goals.md` — **read-only для агента**.

> Риск: #76

## Формат

```markdown
# Project Rules

## Code Style
- Используй функциональные компоненты
- Максимум 100 строк на функцию

## API Changes
- Любое изменение публичного API — документировать в CHANGELOG

## Testing
- Покрывать happy path + 2 edge cases для каждой функции
```
