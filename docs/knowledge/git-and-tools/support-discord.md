## [договорённость] Поддержка и баги через Discord → roadmap

**Контекст:** решение оператора — основной канал поддержки Discord; отдельная тема **`bugs`** (2026-05).

**Суть:** пользовательские баги собираются в Discord (**bugs**); maintainers по **команде** (скрипт/бот) читают выгрузку и добавляют приоритетные пункты в **`docs/roadmap.md`** как `- [ ]`, без автоматического закрытия без human review. Дубликаты сверять с GitHub Issues.

**Применение:** при triage и при проектировании CLI/бота (Bot Token только в env/secret store).
