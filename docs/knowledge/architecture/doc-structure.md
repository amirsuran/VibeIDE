## [архитектура] Структура документации проекта

**Контекст:** создана в первой сессии работы над VibeIDE.

**Суть:** вся документация живёт в `docs/v1/` (33 файла по модулям) + `docs/roadmap.md` (чеклист с фазами) + `docs/idea.md` (исходный документ идеи). Новую сессию начинать с чтения `docs/roadmap.md` — там состояние прогресса через чекбоксы `[x]`.

**Применение:** при любом вопросе «что уже сделано» или «с чего начать» → читать roadmap.md.

## [архитектура] Модуль `contrib/vibeide` и префикс `vibeide.*`

**Контекст:** ребренд после форка CortexIDE; внутренние имена совпадают с продуктом VibeIDE.

**Суть:** дерево **`src/vs/workbench/contrib/vibeide/`**; настройки, storage keys, большинство command ID и IPC (`vibeide-channel-*`) используют префикс **`vibeide`**. Миграция старых **`cortexide.*`** ключей не делалась (внутренние сборки). Ассет онбординга: `src/vs/workbench/browser/media/vibeide-main.png`.

**Применение:** новые фичи только в `vibeide`; Cortex сравнивать клоном в стороне без `git merge` в основную ветку (см. `FORK_CHANGES.md`).
