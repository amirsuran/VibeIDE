Работай по маршруту ниже как по **источнику истины о готовности проекта**.

**Протокол:**
1. Если в проекте ещё нет файла `.vibe/pipeline.md` — создай его с этим маршрутом.
2. Назови **текущий этап**, сделай работу по нему, обнови чекбоксы прямо в `.vibe/pipeline.md` (`[ ]` → `[~]` → `[x]`).
3. Иди строго сверху вниз. Этапы с 🔒 — **блокирующие**.
4. Если просят задеплоить/зарелизить, а хоть один 🔒 выше не `[x]` — **откажись** и ответь ровно: «Стоп. По pipeline.md не закрыты блокирующие этапы: <список>». Предложи закрыть их первыми.
5. Не помечай этап `[x]` без выполненного критерия. «Работает у меня» — не критерий; критерий — воспроизводимая проверка.

## 1. Идея и гипотеза            [ ]
- [ ] Одна фраза: кто пользователь и какую боль закрываем
- [ ] Критерий успеха (метрика) — как поймём, что взлетело

## 2. Архитектура · BaaS-first    [ ]
- [ ] Выбран BaaS (Supabase/Firebase/…), а не свой сервер без реальной нужды
- [ ] Схема данных + модель доступа (права/RLS) продуманы заранее

## 3. Интеграции · доки-first     [ ]
- [ ] Каждая внешняя API — сверена с официальной докой, а не «по памяти»
- [ ] Ключи и секреты в env, не в коде

## 4. Реализация MVP              [ ]
- [ ] Основной happy-path работает end-to-end
- [ ] Обработаны ошибки и пустые состояния

## 5. Сквозные e2e-тесты      🔒  [ ]
- [ ] Ключевой пользовательский сценарий покрыт e2e
- [ ] Тесты зелёные локально и в CI

## 6. Безопасность · атака на прод 🔒 [ ]
- [ ] Авторизация/права проверены попыткой обхода
- [ ] Нет открытых бакетов/эндпоинтов; rate-limit на чувствительном

## 7. Производительность          [ ]
- [ ] Нет N+1 и тяжёлых запросов на горячем пути
- [ ] Проверено на объёме данных, близком к реальному

## 8. Продуктовая аналитика   🔒  [ ]
- [ ] События ключевой воронки шлются (кто, что, когда)
- [ ] Есть место, где это реально смотрят

## 9. Наблюдаемость               [ ]
- [ ] Ошибки собираются (Sentry/аналог)
- [ ] Есть алерт на падение

## 10. Пред-релизный чек-лист      [ ]
- [ ] Бэкапы/миграции обратимы
- [ ] Прогон по этапам 1–9: все 🔒 закрыты

## 11. Релиз в прод           🔒  [ ]
- [ ] Деплой + smoke-проверка вживую
- [ ] Откат отрепетирован
